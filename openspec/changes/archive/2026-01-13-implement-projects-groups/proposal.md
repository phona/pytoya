# Change: Projects & Groups Management

## Why
Users need to organize their invoice processing work into projects and groups for better management and context isolation.

## What Changes
- Create projects CRUD APIs (create, list, read, update, delete)
- Create groups CRUD APIs under projects
- Implement project detail page with groups list
- Add group creation/edit forms
- Build UI components (ProjectCard, GroupCard, ProjectForm)

## Impact
- Affected specs: New project-management capability
- Affected code: New modules in backend and frontend
